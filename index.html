GitSync

Git Sync is a Luau-based developer tool that provides Roblox programmers a simple and efficient way to store and retrieve scripts to and from a cloud-based database (GitHub).

See details
  • Sharing scripts (examples (snippets), templates, portfolios, team projects, coding challenges, etc.)
  • Transferring script/directories between places/games (ex. You want to make a campaign with a consistent gameplay system, so you would otherwise have to either copy and paste the scripts into every place (level) or create a Roblox package and redo everything every time you update the system)
  • Backing up scripts/directories outside of Roblox quickly and easily
  • Saving scripts so that you can view and work on them outside of Roblox Studio on almost any device
  • Quickly interacting with GitHub (save/update/import a file or directory in a few clicks, and get back to coding as soon as possible)

This is a plugin, which means it operates directly inside of Roblox Studio. This makes it a good choice for developers who don't need any functions beyond those offered by the plugin (pushing, pulling, branching, file-viewing) and developers who like having a very quick and easy way to interact with GitHub.

Matrix Datatype

This project is a pure Lua matrix datatype I created while self-learning Linear Algebra.

See details

I learned about vectors in Precalculus, and they reminded me of the Vector libraries I often see in game development. I investigated further, and learned about the applications and methods of representing vectors. The topics that interested me most were matrix representation and graphics programming, so I did a deep-dive into Linear Algebra.

This datatype is fast for the most part. Properties and methods include: ClassName, Size, Contents, assign, flood, fill, replace, enumerate, transpose, getDeterminant, getRank, getArea, getDimensions, getInverse, getTrace, getIdentity, Zero, One, isSymmetric, isSingular, isInvertible, and find. I implemented all operations, including pseudo-division using Moore-Penrose pseudo-inversion for non-square matrices.

PyLua Uroboros Quine

An Ouroboros program that runs in Lua and Python.

See details

In programming, an Ouroboros is a program that outputs a program in a different language that will output the initial program. In this case, I have a Lua program "A" that when run will produce Python program "B." When Python program "B." is run, it will produce Lua program "A." There is an interesting project where someone used 128 languages in an Ouroboros. This concept is related to "bootstrapping," which is a common way to kickstart self-compiling languages.

BrainF*** Interpreter

A Lua-based program that interprets and runs BF code with a user-friendly CLI.

See details

BrainFuck is an esoteric programming language, which means it is meant to fulfill the bare-minimum requirements needed to be considered Turing-Complete. The language has 8 commands, which makes creating an interpreter less of a hassle than it is for most other languages. This pure Lua BF interpreter was a personal project I set out to complete after learning about the language at a Robotics meeting one night.
