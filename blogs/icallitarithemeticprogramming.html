Background

What I'm going to discuss is boolean math. Particularly, we're going to discuss how to use math like a coding language! I'm not going over any "history of booleans," and I couldn't really find what this was called online, but I'm sure it's already been explored.

I found out about this while learning about C.P. Willans' prime factory. The formula is highly inefficient, but it's a good demonstration of how we can use math as a programming language. In an effort to dissect the formula, I delved into online research and discovered a lot of logic I find exciting as a long-time coder and math enthusiast.


Boolean Algebra

Let's start simple. The way boolean logic works is that we have integers a and b, which can each be either 0 or 1. The logical operators AND, OR, NOT, and XOR all return 0 or 1 as well.

We're going to assume this widely accepted idea that 0 represents false and 1 represents true.

Where a ∈ {0, 1} and b ∈ {0, 1}

  • AND is represented by a × b
  • OR is represented by a + b − ab
  • NOT is represented by 1 − a
  • XOR is represented by a + b − 2 × a × b

Let's look at deriving these formulas, though. I promise it's intuitive. From now on, I'll be using 0 interchangeably with false and 1 interchangeably with true.


AND Operator

The simplest operator is AND. In boolean logic, AND is a 2-input function that returns true if and only if both inputs are true. Otherwise, it's false.

f(a, b) = 1 ⇔ a = 1 ∧ b = 1

Thus, the operator can also be represented by multiplying the two inputs, since this process will only return 1 if both values are 1 and not 0.

f(a, b) = a × b
