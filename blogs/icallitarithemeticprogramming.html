Background

What I'm going to discuss is boolean math. Particularly, we're going to discuss how to use math like a coding language! I'm not going over any "history of booleans," and I couldn't really find what this was called online, but I'm sure it's already been explored.

I found out about this while learning about C.P. Willans' prime factory. The formula is highly inefficient, but it's a good demonstration of how we can use math as a programming language. In an effort to dissect the formula, I delved into online research and discovered a lot of logic I find exciting as a long-time coder and math enthusiast.

If you'd like to skip the boolean algebra introduction and get to the good stuff, click here!



Boolean Algebra

Let's start simple. The way boolean logic works is that we have integers a and b, which can each be either 0 or 1. The logical operators AND, OR, NOT, and XOR all return 0 or 1 as well.

We're going to assume this widely accepted idea that 0 represents false and 1 represents true.

Where a ∈ {0, 1} and b ∈ {0, 1}

  • AND is represented by a × b
  • OR is represented by a + b − a × b
  • NOT is represented by 1 − a

Let's look at deriving these formulas, though. I promise it's intuitive. From now on, I'll be using 0 interchangeably with false and 1 interchangeably with true.



AND Operator

One of the simplest operators is AND. In boolean logic, AND is a 2-input function that returns true if and only if both inputs are true. Otherwise, it's false.

f(a, b) = 1 ⇔ a = 1 ∧ b = 1

Thus, the operator can also be represented by multiplying the two inputs, since this process will only return 1 if both values are 1 and not 0.

f(a, b) = a × b

NOT Operator

AND may be simple, but NOT is by far the simplest operator we're going to talk about. It is a 1-input function that toggles the input, returning the opposite of its original value. NOT true is false, and NOT false is true.

f(a) = 1 − a

The truth table here is comically short:

A

OUTPUT
1 0
0 1


OR Operator

OR isn't complicated either. OR is another 2-input function, and it returns true if and only either or both of the inputs are true. If nothing is true, then OR returns false.

f(a, b) = 1 ⇔ a = 1 ∨ b = 1

And from that information, we can derive this formula:

f(a, b) = a + b − a × b

But WHERE IN THE WORLD did this formula come from? Well the OR operator is only false in one scenario, and that's when neither a nor b is true. We can get this condition by writing NOT A AND NOT B. Then, we invert this condition by using NOT again. Our final result is NOT ((NOT A) AND (NOT B)). If you simplify, you'll get the equation mentioned above.

f(a, b) = 1 − ( (1 − a) × (1 − b) )

= 1 − ( 1 − a − b + a × b )

= 1 − 1 + a + b − a × b

= a + b − a × b

Finding XOR

The XOR operator is short for "exclusive or," which is only true if either a or b is true (but not both simultaneously). I'll give you the opportunity to try this one on your own and check your work. When we solved for a function representing the OR operator, we used the truth table and our existing knowledge of AND and NOT to figure things out. Similarly, we can use our understanding of the AND, OR, and NOT operators to derive a formula for XOR.

XOR is true if and only if a is true AND b is false, or a is false AND b is true. From there, we derive an expression and simplify.

I encourage you to check the work yourself, and as you do, remember the new rules we just introduced.

f(a, b) = (a × (1 − b) ) + (b × (1 − a) ) − (a × (1 − b) ) × (b × (1 − a))

= (a − a × b) + (b − a × b) − a × b × (1 − a) × (1 − b)

= a + b − 2 × a × b − a × b × (1 − a) × (1 − b)

= a + b − 2 × a × b − a × b × (1 − b − a + a × b)

= a + b − 2 × a × b − a × b + a × b2 + a2 × b − a2 × b2

= a + b − 2 × a × b − 2 × a × b − a × b + a × b + a × b − a × b

= a + b − a × b − a × b − a × b + a × b + a × b − a × b

= a + b − 2 × a × b
