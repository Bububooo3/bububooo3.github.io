Background

What I'm going to discuss is boolean math. Particularly, we're going to discuss how to use math like a coding language! I'm not going over any "history of booleans," and I couldn't really find what this was called online, but I'm sure it's already been explored.

I found out about this while learning about C.P. Willans' prime factory. It's highly inefficient, but it's a good demonstration of how we can use math as a programming language. In an effort to dissect the formula, I delved into online research and discovered a lot of logic I find exciting as a long-time coder and math enthusiast.

Operators

Let's start simple. The way boolean logic works is that we have integers a and b, which can each be either 0 or 1. The logical operators AND, OR, NOT, and XOR all return 0 or 1 as well.

We're going to assume this widely accepted idea that 0 represents false and 1 represents true.

  • AND is represented by ab
  • OR is represented by a + b - ab
  • NOT is represented by 1 - a
  • XOR is represented by a + b - 2ab

But where do these come from? Well, it begins with the AND operator and the NOT operator, which are both pretty intuitive. 1 times 1 means both a and b are true, and any other combination returns 0 aka false.
